a fork is a copy of a repository. 
Forking a repository allows you to freely experiment with changes 
without affecting the original project.

git commands 
+++++++++++++++++

to stage files for commit: 
1) git add filename.experiment
2) git add . ( for all the changes to be staged)

3) git commit -m "message" (commit to the local repo. you are ahead of the remote repo, therefore, you need a push)
4) git push

starting from remote repo
+++++++++++++++++++++++++++++++++

1) create a fork which is like a copy of the remote where you can do the changes
2) git clone <url of fork> 
3) git add . (if you make any changes in the files of the repo)
4) git commit -m "commit message"
5) git push


differences in git
+++++++++++++

git diff <file>
git diff 
git diff --cached


starting from local repo
++++++++++++++++++++++++++++++++++
You have a folder on your local repository which you want to connect to a 
git remote repository

1) create the local repository
2) create the remote repository
3) git add . (to add all the files of the local repo)
4) git remote add origin <url of the remote repo>
5)git push --set-upstream origin main (git push only would not work in this case )
